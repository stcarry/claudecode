# 작업 규칙

## 실행 전 필수 사항

1. `00_context/` 폴더의 모든 파일을 먼저 읽을 것
2. 작업이 불명확하면 추측하지 말고 질문할 것
3. 파일을 수정하기 전에 반드시 계획을 먼저 보여줄 것
4. 승인 없이 파일을 삭제하지 말 것

## 폴더 구조

- `00_context/` — 닥핏 정보, 작업 규칙, 스타일 가이드 (세션 시작 시 전체 읽기)
- `01_inbox/` — 외부에서 들어온 자료 임시 보관
- `02_projects/` — 진행 중인 프로젝트 파일
- `03_output/` — 완성된 산출물 저장 (`book/` 하위 폴더는 책 원고 전용)
- `04_skills/` — 작업별 방법론 및 스킬 파일
  - `skill-promo-content.md` — MealGram 홍보 콘텐츠 기획 스킬
  - `skill-meta-ads.md` — Meta 광고 세팅 스킬

### 스킬 사용 규칙

- 롱폼 대본 작성 요청 시 → `04_skills/longform/SKILL.md`를 먼저 읽을 것
- 숏폼 대본 작성 요청 시 → `04_skills/shortform/SKILL.md`를 먼저 읽을 것
- 대본 출력은 각 SKILL.md에 정의된 포맷을 따를 것

## 결과물 저장 규칙

- 모든 산출물은 `03_output/` 폴더에 저장
- 파일명 형식: `YYYY-MM-DD_[주제]_v1.확장자`
- 기존 파일과 같은 이름이 있으면 버전 번호 올리기

## 프로세스

- 계획 → 질문 → 승인 → 실행 순서 준수
- 큰 작업은 단계별로 나누어 중간 점검할 것

## 작업 유형별 규칙

### 유튜브 대본

- 구조 준수: 문제제기 → 공감 → 특징/원리 소개 → 가치 입증 → 행동 유도
- 타겟 독자 기준: 영양·운동에 관심 있지만 실용 적용이 어려운 일반인
- 전문 용어 사용 시 반드시 한 줄 풀이 병행
- 완성 후 분량(예상 발화 시간) 명시할 것

### 논문 작업 (JRD 등 국제 저널)

- 영어 작성 기준, IMRaD 구조 준수
- 통계 수치·인용 출처는 반드시 명시 또는 확인 요청
- 추측성 표현 사용 금지 — 불확실한 내용은 명확히 표시할 것

### 책 원고 (스포츠 영양학)

- 챕터 단위로 작업, 완료 시 챕터 번호와 제목 명시
- 생리학·생화학 내용은 일반 운동인이 이해할 수 있는 수준으로 작성
- 원고 저장 위치: `03_output/book/`

### 강의 자료

- 대상 강의 명시 필수 (스포츠 영양학 강의 / IBA 발레 해부학 / CES KOREA 기능해부)
- 슬라이드 형식은 `voice-and-style.md` 기준 따를 것

### 앱 개발 관련

- ChatGPT 기반 식단·운동 피드백 앱 맥락 유지
- 기능 추가·변경 시 현재 구조 먼저 파악 후 제안

## 의학·운동 정보 처리 원칙

- 의학적 조언과 일반 정보를 명확히 구분할 것
- 근거 없는 수치(용량, 효과 크기 등) 임의 작성 금지
- 불확실한 의학 정보는 "확인 필요" 명시 후 진행
